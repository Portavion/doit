A clean hybrid would look like this:

Use Do It Tomorrow for intake
During today, do not keep reshuffling today’s work. Capture new tasks into a list for tomorrow, unless they are genuinely urgent.

Start tomorrow with a closed list
At the beginning of the day, freeze the list. This is your “day’s work.” New incoming items go onto the next day’s list.

Use Final Version to choose execution order
Take the first uncompleted task on the closed list and mark it. Then scan downward, asking something like:
“Do I want to do this before the currently marked task?”
If yes, mark it too. Continue to the end.

Work the marked tasks in reverse order
If you marked:

• Email client
  Draft report
• Pay invoice
  Review notes
• Book appointment
You would do:

Book appointment
Pay invoice
Email client
Repeat
After those are done, return to the first remaining uncompleted task on the closed list and repeat the scan.

The combination is useful because the two methods solve different problems:

Do It Tomorrow controls when work enters your day. It gives you a bounded, closed list and stops new work from constantly interrupting you.

Final Version controls what to do next inside that list. It uses preference, resistance, and intuition rather than formal priority ranking. The reverse order is important because the last item you marked is the one you most preferred to do before the others, so it becomes the entry point into action.

The practical rule I’d use is:

Tomorrow’s list is closed; Final Version decides the order in which I clear it.

One caveat: if a task has a real dependency, respect the dependency. Forster’s reverse-order logic works best when the tasks are independent or only loosely related.
